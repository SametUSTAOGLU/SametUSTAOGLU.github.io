AWS CLI

AWS CLI bir komut satırı ara yüzüdür. Aslında extra bir program veya paket olarak çalışmaz. Yüklendiğinde windows kullanırsanız powershell, linux kullanıyorsanız bash veya alternatifi herhangi bir shell, Mac kullanıyorsanız zshall ile CLI'ı kullanabilirsiniz. Hatta windows içerisinde Ubuntu için kullanılan WSL'de dahi yükleyip kullanabilirsiniz.Yüklenme işlemi çok basit olduğu için burada anlatmayacağım. Aşağıdaki linkte AWS her işletim sistemi için adım adım anlatmış takılacağınız bir durum olmayacaktır. Sadece şuna dikkat ediniz 2 adet sürümü mevcut CLI ve CLI2. CLI2 yüklemenizi tavsiye ediyorum daha kabiliyetli kendisi. AWS'te Amazon Linux 2 AMI'si ile bir instance ayağa kaldırdığınızda hala CLI2 değilde CLI görebilirsiniz. Böyle bir durumda önce CLI'ı silmeniz daha sonra CLI2'yi yüklemelisiniz. Ayrıca yükleme işlemini AWS kendi sitesinden yapmanızı öneriyor.

Neden kullanıyoruz onu açıklamaya çalışalım. AWS konsol ile yapabileceğiniz tüm işlemleri ulaşabileceğiniz tüm kaynaklara CLI üzeriden ulaşabiliyoruz. Hatta konsoldan müsade edilmeyen birçok işlemi CLI üzerinden halledebiliyoruz. Herhangi bir terminal ile başka hiçbir şeye gerek kalmadan kaynaklarınızı yönetebilir ve izleyebilirsiniz.

Yüklemeden sonra başarılı olup olmadığını versiyon kontrolü ile yapabilirsiniz. 

aws --version

Kurulumu yaptıktan sonra konfigurasyon yapmalısınız. Bunun için AWS hesabı oluştururken veya IAM user oluştururken "Programmatic Access" seçeneğini seçtiyseniz aldığımız Access key ID ve Secret access key'e ihtiyacınız olacak. Kaybetti iseniz veya hiç almadıysanız IAM servisinde My security credentials menüsünden yenisini oluşturabilirsiniz.

samet@lenovo:~$ aws configure
AWS Access Key ID [****************E665]: 
AWS Secret Access Key [****************Uq+0]: 
Default region name [us-east-1]:               >> Size yakın region seçiniz.
Default output format [yaml]:                  >> json, yaml, text, table

aws configure komutu verdiğimizde bizden id, key, default region ve default output format bilgileriniz isteyecek. Ayrıca aws configure import --csv file://credentials.csv komutu ile csv dosyası göstererek de konfigüsasyon yapılabilir.

aws help komutu ile takıldığınız yerde yardım alabilirsiniz.

Yukarıdaki ayarlar ile belirlenen user için işlemler yapılır ayrıca baş user ile de işlem yapılmak istenirse "aws configure --profile diger-kullanici" komutu ile yeni konfigurasyon yapılması gerekir. diiger-user ile verilmek istenen tüm komutların arkasına aşağıdaki gibi ekleme yapılmalıdır.

aws s3 ls -profile diger-kullanici

CLI'da komut yapımız aşağıdaki gibi olacaktır. Tüm komutlar AWS ile başlar ve devamında kaynak ismi yazılır.

Örnek olarak kayıtlı iam user ları listeleyen bir komut yazalım.

aws iam list-users

Diyelim kullanıcı listeleme komutunu hatırlayamadınız o zaman da;

aws iam help >> ile yardım alabilirsiniz seçenekleri size listeleyecek.

Farklı servislerle biraz uygulama yapalım

Yeni bir kullanıcı oluşturalım. Konsoldan teyit edelim

aws iam create-user --user-name cli-user

Şimdi de bir grup oluşturalım daha sonra cli-user kullanıcısını o gruba ekleyelim.

aws iam create-group --group-name cli-grup

Kullanıcının şifresini değiştirelim.

Şimdi S3 servisi ile ilgili komutlara bakalım.

Senaryomuz şöyle olsun Önce bucket oluşturalım. Local'den Bucket'ın içine dosya ve klasör atalım, aynı şekilde konsoldan yüklediğimiz dosyaları local'e alalım. Sonra içindekileri listeleyelim ve en son bucket'ı silelim.

Öncelikle cli-bucket-samet isimli bucket oluşturalım.

aws s3 mb s3://cli-bucket-samet

Şimdi de local'imizdeki dosya ve klasörleri bucket'a kopyalayalım. Daha sonra da klasörün içindekileri gönderelim. Bunu kopyalama veya taşıma şeklinde yapabiliriz ama taşıma yapılırsa dosyalar kaynaktan tekrar ulaşılamaz. Kopyalama için cp, taşıma için mv komutları kullanılır biz bura kopyalama yapacağız.

Önce dosya gönderelim.

aws s3 cp dosya-1 s3://cli-bucket-samet

Eğer dosyanın ismini hedef bucket'da değiştirmek isteseydik bucket isminden sonra /yeni_isim şeklinde ilave edecektik.

Şimdi de yukarıda gördüğümüz folder isimli klasörü gönderelim.

aws s3 cp folder s3://cli-bucket-samet/s3folder --recursive

--recursive parametresi kullandık s3folder isimli klasöre kopyaladık ismini değiştirdik. Eğer s3'ten sonra klasör ismi girmese idik direk tüm dosyaları bucket içine atardı yeni klasör oluşturmadan. Şimdi cli kullanarak yüklediklerimiz listeleyelim.

Listeleme için ls komutu kullandık. Eğer ls den sonra herhangi bir değer girmese idik tüm bucket'ları listelerdi. PRE ifadesine dikkat edelim folder önüne prefix kısaltması olarak koyuyor. Dosyalarda ise oluşturulma zamanı boyutu ve ismini gördük. Biz listelerken folder içi de gözüksün deseydik aşağıdaki komutu kullanacaktık. - -recursive parametresinden sonra -- summarize parametresi de kullansaydık içierisndeki dosya sayısı ve toplam boyut bilgisini de alabilirdik.

Şimdi de konsoldan upload yapalım ve dosyayı cli ile local'imize indirelim. Aşağıda kutu içindeki dosyayı local'e alacağız.aws s3 cp s3://cli-bucket-samet/konsoldan-yuklendi .

aws s3 cp s3://cli-bucket-samet/konsoldan-yuklendi .

"." ile bulunduğumuz klasöre aldık istersek yol belirtip istediğimiz yere alabilirdik.

cp yerine sync komutu kullansa idik aynı dosyalar hedefte de varsa boyutu farklı olanları alacaktı. Ayrıca bu kopyalama bir bucket'dan diğer bucket'a da olabilirdi.Önce farklı region'da bucket oluşturalım daha sonra içeriği kopyalayalım. Bu örneği yapalım farklı regionda bucket oluşturalım ve kopyalama yapalım.

aws s3 cp s3://cli-bucket-samet s3://cli-diger-bucket --recursive

Şimdi de EC2 üzerinde cli üzerinden örnekler yapalım. Şöyle bir örnek olsun. Bir EC2 çalıştıralım. Daha öncesinde bir key-pair oluşturalım. Sonra EC2'nun özelliklerine bakalım. Son olarak da EC2'yu önce stop, sonra start, daha sonrada terminate edelim.

Önce key-pair oluşturalım.

aws ec2 create-key-pair --key-name yeniKey

Oluşturduğumuzda private key'i bize gösteriyor.

Şimdi de bir EC2 ayağa kaldıralım.

aws ec2 run-instances --image-id ami-01cc34ab2709337aa \

--count 1 --instance-type t2.micro --key-name yeniKey --security-groups ec2-icin

ami'yi konsoldan baktım 🙂. count ile kaç adet oluşturacağımızı yazdık. instance type'ını, yeni oluşturduğumuz key'i ve daha önce oluşturduğumuz ec2-icin isimli security grubu kullandık. Çıktı olarak çok uzun bir sonuç döndürdü. Diyelim ki bize tüm instance'ların id'si ve public adresleri lazım. Bunu sadeleştirebiliriz aşağıdaki şekilde.

aws ec2 describe-instances \

--query "Reservations[].Instances[].{Instances:InstanceId,IPv4:PublicIpAddress}"

Şimdi çalışan bir ec2'yu stop eddelim.

aws ec2 stop-instances --instance-ids i-0fc165951c433a4db

yukarıda bazı kodlar var. Bu kodlar instance'ın durumunun kodları.

0 > pending

16 > running

32 >shutting down

48 > terminated

64 > stopping

80 > stopped

Şimdi tekrar çalıştıralım.

aws ec2 start-instances --instance-ids i-0fc165951c433a4db

Pending'e geçmiş önceki pozisyonu stopped imiş.

Şimdi de terminate edelim bitirelim.
